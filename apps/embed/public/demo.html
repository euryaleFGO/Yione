webLing 嵌入 SDK · 接入演示

这个页面本身就是"第三方网站"。下面三个 section 展示三种接入方式。

1 · Script tag（最常用，一段脚本接入）

本页右下角那个紫色按钮就是它 —— 点击展开会弹出 Live2D 玲和聊天面板。 

<script
  src="https://<webling-host>/embed/embed.js"
  data-api-key="demo"
  data-character-id="ling"
  data-position="bottom-right"
  async defer
></script>
本演示页直接 <script src="/embed/embed.js" data-api-key="demo">，用 backend dev 模式的回退 tenant（任意 api_key 都通过，方便本地调试）。

2 · <web-ling> Custom Element（声明式，同样 CDN）

加载 web-component.js 后，写一个 HTML 标签就能嵌入。加 embedded 属性会把 iframe 直接渲染在自身位置（下方示例）。 

<script src="https://<webling-host>/embed/web-component.js" defer></script>
<web-ling
  api-key="demo"
  character-id="ling"
  base-url="https://<webling-host>"
  embedded
></web-ling>
如果不加 embedded，行为等同于 Script tag，右下角悬浮按钮 + 可展开面板。

3 · iframe 直接嵌入（最隔离）

不想加载任何 JS 也可以，直接把 iframe 指到宿主的 embed.html。 postMessage 协议可以和父页面通信（见下方 README）。 

<iframe
  src="https://<webling-host>/embed/embed.html?api_key=demo&character_id=ling"
  allow="microphone; autoplay; clipboard-write"
  style="width: 380px; height: 620px; border: 0;"
></iframe>

父子通信（postMessage）

所有形态共用同一套合约：target: 'webling'（父→子）、 source: 'webling'（子→父）。 

// 父 → iframe
iframe.contentWindow.postMessage(
  { target: 'webling', type: 'send', text: '你好' },
  '*',
);

// iframe → 父
window.addEventListener('message', (e) => {
  if (e.data?.source !== 'webling') return;
  // e.data.type: ready | session | state | message | error | closed | request_close
});
